#仕様:

  サイトの説明:
  このサイトは、近年人工知能(AI)の技術の向上が著しく、ある研究によると2045年には人間の能力を超えるシンギュラリティ(技術的特異点)を迎えることになる。というネット記事を見て、いまいち現実味を感じなかったので何かAIの機能を使って、それを身近に感じられるような体験が作れたら面白いと思いました。そうした背景から、人間がAIと会話をして、その返答をネガポジ判定して好感度を上げるゲームという形にしました。サイトの成功としては、このゲームをして、頑張って好感度を上げるためにAIの癖をついて言葉を考えて入力するという行動によって、AIに対しての今までの立場を揺るがすような体験をもたらすことだと思います。

  簡単なルール説明:
  ・このゲームは人間と人間による1対1のオンライン対戦ゲームです
  ・しかし、直接なんらかのステータスで戦うわけではありません
  ・プレイヤーはそれぞれが自分の会話相手となるAIと5ターンの会話を行います
  ・会話内容からネガポジ判定を行い、好感度とします。->それがこのゲームのステータスです
  ・5ターンを終えると対戦相手のステータスがわかり、勝敗が決まります。以上

  画面構成*
  トップ画面     : ゲームの説明、ログイン、新規登録機能
  ゲーム画面     : 実際のゲームができる画面(ゲスト、ログインプレイヤーともに実装済み)
  マイページ画面  : ログインしているユーザーが自身の戦歴を見ることができ、プレイヤーとしてゲームを行うことができる

  また、ログイン情報はセッションを用いて保存しているため、ログアウトしない限りはサイトにアクセスすると、自動的にマイページに遷移する。


#デザインコンセプト:
  今回のオリジナルwebサイトのデザインは色使いにこだわりました。シンギュラリティという近未来的なコンセプトから、カラフルでちょっと暗めの配色を用いて近未来感を出しています。またゲーム画面ではディーラー・プレイヤー・相手プレイヤーの3者を三つの色で表現することで、わかりやすい差別化を図りました。わかりにくいですが、1ターンごとに、スコアによって背景色が寒色から暖色に寄っていきます。これはAIの好感度が上がっている状態を、スコアを見せずに視覚的に判断できるという工夫です。チャット風であるゲーム画面はLINEのようにテキストを上に逐一表示できるような形を目指して作成しました。ユーザー待ち時間、ゲームの終了等も動的にディーラーのメッセージが逐一表示されるようになっています。

#動作について:
  このゲームはユーザーとユーザーのやり取りはすべてwebsocketを利用していて、遅延がほとんどなくやり取りができています。
  サーバーとクライアントのやり取り、スコアの更新等はajaxを用いて行っています。

  *今回、ネガポジの判定をAPIにさせるつもりでしたが、予定していたAPIがAI自身の発言はネガポジ判定できないことに気づき、自作することにしました。極性判断等ネットで調べたものを元に作りましたが、自作のためポジティブ度を独断と偏見で決めています。
  また、テストのため、語彙が少ないですが、これから徐々に増やしていく予定です。

#作成してみて:
  今回、STEP3の最終課題でオリジナルサイトを作成するにあたって、webソケットを用いたやり取りや、webAPIを使うことなど初めて使用する技術があり、それぞれをまず勉強するところから始めたが、それによって今まで知らなかったweb技術の理解が深まりました。ajaxを用いた際に起こる、非同期通信のズレなどに悩まされたり、webapiの使用時にクロスドメイン対策に引っかかるなど、ネットで調べて実際に動かしてみてやっと意味がわかることが多くあり、いい体験ができたと思います。また、開発にあたってのスケジュール計画をする力もついたと思います。これまで締め切りがある中でwebアプリやサイトを作る経験がなく、今回のオンラインスキルアップで3回に渡ってそんな経験ができたことで、優先順位をつけることの大切さ、進行状況を常に管理して作業を見積もることなどに、多くの意味を感じました。実際の実務ではクライアントさんの要望等を聞いて機能やデザインを実装して、締め切りに間に合うようにいいものを作らなければいけないので、まだまだ自分の体験はそれに比べると楽なものだとは思いますが、少しでもそんな実務のプログラミングをイメージできたので、いい体験になったと思います。また、これを機にフロントエンド、バックエンドともに自分で設計してサイトを作る流れを理解できたので、今までアイデアはあったけど作れなかったサイトを作っていきたいです。
